# 📊 Monitoring in DevOps

> **Level:** Beginner → Intermediate
> **Phase:** 1 - DevOps Fundamentals

---

# 📖 Introduction

Imagine you've successfully deployed your application to production.

Thousands of users are using your website.

Suddenly...

Customers start complaining:

* Website is slow
* Login is failing
* Payment isn't working
* Website is down

How do you know what's happening?

This is where **Monitoring** comes in.

Monitoring helps engineers observe the health, performance, and availability of systems in real time.

---

# What is Monitoring?

Monitoring is the continuous process of collecting, analyzing, and visualizing data from applications, servers, networks, and cloud infrastructure.

It helps identify problems before users are affected.

---

# Simple Definition

**Monitoring is the process of continuously tracking the health and performance of systems, applications, and infrastructure.**

---

# Why Monitoring is Important?

Without monitoring:

```text
Application Down

↓

Nobody Knows

↓

Customers Complain

↓

Business Loses Money
```

With monitoring:

```text
Application Down

↓

Monitoring Detects Problem

↓

Alert Sent

↓

Engineer Fixes Issue

↓

Application Back Online
```

# Types of Monitoring

## Infrastructure Monitoring

Monitors servers.

Examples:

* CPU
* RAM
* Disk
* Network

---

## Application Monitoring

Monitors application performance.

Examples:

* Login Time
* API Response Time
* Error Rate

---

## Network Monitoring

Monitors:

* Latency
* Packet Loss
* Bandwidth

---

## Log Monitoring

Monitors application logs.

Example:

```text
INFO Server Started

WARNING High Memory Usage

ERROR Database Connection Failed
```

---

# Metrics

A **Metric** is a numerical value collected over time.

Examples:

| Metric        | Example |
| ------------- | ------- |
| CPU           | 45%     |
| RAM           | 70%     |
| Disk          | 60%     |
| Response Time | 250 ms  |

---

# Logs

Logs are records of events generated by applications.

Example:

```text
2025-06-28 10:15:12

User Login Successful

User: admin
```

Logs help troubleshoot issues.

---

# Alerts

Alerts notify engineers when something is wrong.

Example:

```text
CPU > 90%

↓

Send Email

↓

Slack Notification

↓

SMS
```

# Prometheus

Prometheus is an open-source monitoring system.

Functions:

* Collect Metrics
* Store Time-Series Data
* Query Metrics
* Send Alerts

Architecture:

```text
Server

↓

Node Exporter

↓

Prometheus

↓

Grafana
```

---

# Node Exporter

Node Exporter collects Linux server metrics.

Examples:

* CPU
* RAM
* Disk
* Network

Prometheus reads these metrics.

# Hands-on Practice

## Step 1

Launch Ubuntu EC2.

---

## Step 2

Install Node Exporter.

```bash
wget https://github.com/prometheus/node_exporter/releases/latest/download/node_exporter.tar.gz
```

Extract:

```bash
tar -xvf node_exporter.tar.gz
```

Run:

```bash
./node_exporter
```

---

## Step 3

Install Prometheus.

Configure:

```text
prometheus.yml
```

Add Node Exporter target.

---

## Step 4

Start Prometheus.

Browser:

```text
http://localhost:9090
```

---

## Step 5

Install Grafana.

Browser:

```text
http://localhost:3000
```

---

## Step 6

Connect Grafana with Prometheus.

Create Dashboard.

Visualize:

* CPU
* Memory
* Disk
* Network
